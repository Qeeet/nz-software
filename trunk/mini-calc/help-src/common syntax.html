Any input that isn't a command is considered a mathematical expression (e.g. "sin(x)*pi") and will be calculated and the answer outputted to the screen.
The currently implemented commands are:
  • delete: x - Deletes the function or variable named (x).
  • clear - Clears the output screen.
  • f(x,y,...) = z - Creates a new expression-based function named (f) with parameters (x), (y), etc. The function value (expression) will be (z), which may reference the parameters as variables (e.g. "x^2 + y^2")
  • x = z - Creates a new variable named (x) and sets its value to (z). If (x) already exists, its value is set to (z)
The current operations are:
  • + (Addition)
  • - (Subtraction)
  • * (Multiplication)
  • / (Division)
  • % (Modulus/Remainder)
  • ^ (Exponent)
The rules for expressions are:
  • Expressions are comprised of operators, values, grouped sub-expressions, and functions.
  • Simple expressions contain only values (e.g. "2.5")
  • Complex expressions contain values, operators, and/or functions (e.g. "sin(90) + 2.5 ^ x")
  • Expressions may contain grouped sub-expressions by using parentheses (e.g. "(sin(90) + 2.5 ^ x) - (5 + (2 - 5))")
  • Functions can be used by using the function name and expressions for each parameter (enclosed in parentheses and separated by commas) (e.g. "1 + min(2,3,pi,4 ^ 2)")

Note: To use the results of the previously evaluated expression, use the variable "ans"

Note: In the case of a syntax error, the outputted error information will contain the zero-based index of the character that caused the error.
